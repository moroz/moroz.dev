開發軟體像做蛋糕一樣。
一開始，只有一個粗略的，很抽象的描寫，如：「做蛋糕」。
然而，如果只提供這麼簡約的描寫，恐怕沒有特別多人做得出蛋糕，更不可能每一次都是同一個口味，也無法保證每一次都做得一樣好。

因此，目標要設得更明確，如：「我要做一個蘋果派」。
這樣，我們可以將做蘋果派的流程分得更細：

* 買原料、
* 做蘋果泥、
* 做麵團、
* 將蘋果泥塗在麵團上、
* 烘焙、
* 冷卻。

這時候，一個人看到以上步驟，說不定已經可以做得出還不錯的蘋果派。
那是因為人的大腦會在處理過程中加上大腦內部現有的輸入資料，也就是說每一個人的教育與經驗。
然而，電腦不會自己學習，你要它做什麼，它就做什麼，不會突暫停來思考你所輸入的程式是否合理。

所以上面的每一步都要繼續拆解，將每一個小任務都分為更小的動作。
比如說，做麵團，就是要拿一個金屬碗，將三杯麵粉加進去，加上150克奶油、一杯糖、一茶匙蘇打粉、五克鹽，攪拌均勻。
每一個動作都要分解為不能再拆解的小步驟，才可以保證之後的實作都會一樣可靠。

開發軟體的概念很接近。
每一個功能都要拆成很細節的小步驟，而且還要了解動作之間的相依關係。
比如說，當你開發用電子信箱與密碼登入的功能的時候，可以寫類似這樣的函數：

```go
/// AuthenticateUserByEmailPassword 將查詢資料庫中有沒有電子信箱與密碼
/// 都符合所提供的 `email` 與 `password` 的用戶資料。
func AuthenticateUserByEmailPassword(db *sqlx.DB, email, password string) (*User, error) {
    // 按電子信箱搜尋用戶
    user := GetUserByEmail(db, email)
    // 如果用戶不存在，返回錯誤訊息
    if user == nil {
        NoUserVerify() // 等待幾秒再返回錯誤訊息，加強安全性
        return nil, errors.New("找不到用戶")
    }

    // 如果所輸入的密碼正確，那就成功登入了
    if CheckUserPassword(user.PasswordHash, password) {
        // 返回用戶資料，無錯
        return user, nil
    }

    // 返回錯誤
    return nil, errors.New("您所輸入的密碼不正確")
}
```

```go
func AuthenticateUserByEmailPassword(db *sqlx.DB, email, password string) (*User, error) {
```

從這一行可以猜測 Go 語言宣告函數的語法：

1. 關鍵字 `func`。
2. 函數名稱：通常使用拉丁字母與阿拉伯數字。如果名稱的第一個字符為大寫拉丁字母（`A-Z`），該函數就會變成公開函數，也就是說，這個軟體包（英：_package_）以外的程式碼可以呼叫這個函數。
   Go 語言的名稱可以用中文，但要在前面加大寫拉丁字母才可以當成公開函數。
3. 左小括號 `(`。
4. 零或更多個參數。每一個參數都得指定名稱與類型，但如果兩個以上的參數都是同一個類型，可以在最後一個同類型的參數名稱後再宣告類型。
   `(email, password string)` 同 `(email string, password string)`。
5. 右小括號 `)`。
6. 返回值類型。Go 語言的函數可以返回零或更多的值，所以很多時候任何可能出錯的函數都會返回兩個值：成功的結果與錯誤值。
   上方範例中的 `AuthenticateUserByEmailPassword` 也利用這個技巧，返回值分別為 `*User`（`User` 結構的指標，惹為空指標則未登入成功）與錯誤資訊（進一步解釋錯誤原因，可為空值）。
   如果沒有返回值，這一部分要跳過喔。
7. 左花括號 `{` 後面就是這個函數的內容。
