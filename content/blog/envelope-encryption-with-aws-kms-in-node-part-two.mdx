### Set up AWS CLI

The easiest way to set up AWS credentials in development is using [static access keys](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_credentials_access-keys.html).

Sign in to the AWS Web console. In the IAM dashboard, create a user with at least the `AWSKeyManagementServicePowerUser` policy attached. Then, create an access key for this user. Add this access key to `aws-vault`:

```bash
$ aws-vault add kms-tutorial
Enter Access Key ID: AKIAIOSFODNN7EXAMPLE
Enter Secret Access Key: *****************************************
Enter MFA Device ARN (If MFA is not enabled, leave this blank): [Enter]
Added credentials to profile "kms-tutorial" in vault
```

Create a configuration section for the newly created profile in `~/.aws/config`. You may want to swap `eu-central-1` (Frankfurt) for another [AWS region](https://docs.aws.amazon.com/global-infrastructure/latest/regions/aws-regions.html). Pick the one which is closest to you.

```ini filename="~/.aws/config"
[profile kms-tutorial]
credential_process=aws-vault exec kms-tutorial --json --prompt=terminal
region=eu-central-1
```

When editing this file, make sure you did not leave out a duplicate section header (`[profile kms-tutorial]`). The file will not parse correctly if you do.

Verify the configuration using `aws sts get-caller-identity`. If you have configured the credential flow correctly, the command should return a JSON response similar to the one below:

```bash
$ aws sts get-caller-identity --profile=kms-tutorial
{
    "UserId": "AIDASAMPLEUSERID",
    "Account": "123456789012",
    "Arn": "arn:aws:iam::123456789012:user/kms-tutorial"
}
```

### Inject environment variables using Mise

Write down the `KeyId` or `Arn`&mdash;you will need to specify the resource identifier in all API calls.
A good way to inject the ID into your application is using environment variables.
In the project root, create a file called `mise.local.toml`:

```toml filename="mise.local.toml"
[env]
AWS_PROFILE = "kms-tutorial"

# .KeyMetadata.KeyId or .KeyMetadata.Arn, as returned by CreateKey
ENCRYPTION_KEY_ID = "..."
```

With this configuration file, your shell should now have access to both variables:

```bash
$ echo $ENCRYPTION_KEY_ID $AWS_PROFILE
63dd51d2-4b2d-44d1-9e96-e69c88af10fd kms-tutorial
```

You should treat `mise.local.toml` just like any other environment (dotenv) file&mdash;assume it may contain sensitive values and should not be committed to version control.

Create a `.gitignore`:

```bash filename=".gitignore"
# NPM packages
node_modules/

# TypeScript artifacts
dist/

mise.local.toml
```

Verify that `mise.local.toml` is ignored:

```bash
$ git check-ignore -v mise.local.toml
.gitignore:7:mise.local.toml    mise.local.toml
```

### Install and configure AWS SDK for JavaScript

In order to call AWS APIs in Node.js, we need to install the [AWS SDK for JavaScript](https://aws.amazon.com/sdk-for-javascript/). In this part, we're only going to need one package, [`@aws-sdk/client-kms`](https://www.npmjs.com/package/@aws-sdk/client-kms). Add it to the project using pnpm:

```bash
$ pnpm add @aws-sdk/client-kms
```

First, let's set up a KMS client instance. Create a new file named `src/kms.ts`:

```typescript filename="src/kms.ts"
import { KMSClient } from "@aws-sdk/client-kms";

const kmsClient = new KMSClient();
```

With the AWS CLI and the `AWS_PROFILE` environment variable correctly set up, the SDK should pick up the correct credentials implicitly.

### Read the Key Encryption Key ID from environment variables

Before we make any calls to the KMS API, we need to first inject the key ID from the environment. We can use a helper function called `mustGetenv` (inspired by Go conventions):

```ts filename="src/kms.ts"
function mustGetenv(name: string): string {
  const value = process.env[name];
  if (!value) {
    console.error(`FATAL: Environment variable ${name} is not set!`);
    process.exit(1);
  }
  return value as string;
}

const ENCRYPTION_KEY_ID = mustGetenv("ENCRYPTION_KEY_ID");
```
