### Inject environment variables using Mise

Write down the `KeyId` or `Arn`&mdash;you will need to specify the resource identifier in all API calls.
A good way to inject the ID into your application is using environment variables.
In the project root, create a file called `mise.local.toml`:

```toml filename="mise.local.toml"
[env]
AWS_PROFILE = "kms-tutorial"

# .KeyMetadata.KeyId or .KeyMetadata.Arn, as returned by CreateKey

```

### Read the Key Encryption Key ID from environment variables

Before we make any calls to the KMS API, we need to know the key identifier in our code. Using [`process.env`](https://nodejs.org/learn/command-line/how-to-read-environment-variables-from-nodejs), we can inject the key ID from environment variables.

For that, I like to use a helper function called `mustGetenv`:

```ts filename="src/kms.ts"
function mustGetenv(name: string): string {
  const value = process.env[name];
  if (!value) {
    console.error(`FATAL: Environment variable ${name} is not set!`);
    process.exit(1);
  }
  return value as string;
}

const ENCRYPTION_KEY_ID = mustGetenv("ENCRYPTION_KEY_ID");
```

### Generate a Data Encryption Key

In this section, we are going to generate a Data Encryption Key (DEK) using the [`GenerateDataKey`](https://docs.aws.amazon.com/kms/latest/APIReference/API_GenerateDataKey.html) API.
The command has two required parameters: `KeyId`, which is the identifier of the Key Encryption Key; and either `KeySpec` or `NumberOfBytes`.

Let's make the call in `src/kms.ts`:

```typescript
import { GenerateDataKeyCommand, KMSClient } from "@aws-sdk/client-kms";

function mustGetenv(name: string): string {
  const value = process.env[name];
  if (!value) {
    console.error(`FATAL: Environment variable ${name} is not set!`);
    process.exit(1);
  }
  return value as string;
}

const ENCRYPTION_KEY_ID = mustGetenv("ENCRYPTION_KEY_ID");

const kmsClient = new KMSClient();

const generateDataKeyResponse = await kmsClient.send(
  new GenerateDataKeyCommand({
    KeyId: ENCRYPTION_KEY_ID,
    KeySpec: "AES_256",
  }),
);
console.log({ generateDataKeyResponse });
```
